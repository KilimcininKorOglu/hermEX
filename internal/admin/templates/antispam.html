Anti-spam

Inbound mail is scored at delivery (SPF, DKIM, DMARC, DNS blocklists, a Bayesian content model, and a SpamAssassin rule subset); a message over the threshold is filed to the recipient's Junk folder.

Greylisting is {{if .GreylistEnabled}}on{{else}}off{{end}}. When on, the first message from an unknown sender (keyed by network, sender, and recipient) is deferred with a temporary failure so a legitimate server retries after a short delay; a spammer that does not retry never gets through. This delays first mail from new senders by a few minutes. Authenticated submission and allowlisted senders are never greylisted.

The quarantine digest is {{if .DigestEnabled}}on{{else}}off{{end}}. When on, each user is emailed a periodic plain-text summary of newly quarantined (Junk) mail, every entry carrying a signed one-click link that releases that one message back to the inbox after a confirmation page. A per-mailbox watermark means only mail new since the last summary is included, and a mailbox with nothing new is skipped.

This also requires a digest_secret in the server configuration, shared by the MTA and webmail; without it the release links do not work.

Outbound abuse limiting is {{if .OutboundEnabled}}on{{else}}off{{end}}. When on, a local account that sends to more than the cap of external recipients within the window has the excess deferred with a temporary failure — a compromised account cannot blast spam unchecked — and an outbound.abuse event is logged once per window so you can disable the account. Authenticated submission only; inbound mail is never affected.
