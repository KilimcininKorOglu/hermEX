TLS certificates

The certificate the gateway (front door) and the mail daemons present. An uploaded certificate overrides the config-file certificate and is picked up within about a minute — no restart. Leave the name blank for the default certificate, or set an SNI host name to serve a specific host. Private keys are stored but never shown again.

Certificate mode

Manual serves the operator-uploaded certificates below. ACME has the gateway obtain and renew Let's Encrypt certificates automatically for every active domain's mail/autodiscover/autoconfig hosts (and the mta-sts host when publishing is on) over the TLS-ALPN-01 challenge, so each name must resolve to the gateway on :443. The apex is excluded — it is the domain owner's own website. A mode change takes effect when the gateway restarts; certificate contents still hot-reload without one.

MTA-STS policy publishing

Publish an MTA-STS policy (RFC 8461) so sending servers require validated TLS when delivering to this server. The gateway serves the policy at mta-sts.<domain>; each domain's detail page then shows the mta-sts, _mta-sts and TLSRPT records its owner must publish. Testing mode reports TLS failures but still delivers; enforce mode makes senders refuse mail to a non-validated MX, so a certificate problem under enforce loses inbound mail. A change applies on the senders' next fetch — no restart.
